TRANSFORMATION · LESSON 07

Named templates, parameters & recursion

Reusable subroutines and the recursion pattern you'll reuse to build ciphers.

So far every rule matched a node. But you also need reusable subroutines — a block of logic you can call with arguments and reuse. That's the named template. And because XSLT 1.0 has no counting loops, repeating work means a template that calls itself. Master this here — the ciphers in Lesson 10 are the same pattern. Folder: examples/06-recursion.

Named templates & parameters

A named template is called explicitly with call-template, passing arguments with with-param:

shape of a named template
<xsl:template name="greet">
  <xsl:param name="who"/>
  <xsl:text>Hello, </xsl:text><xsl:value-of select="$who"/>
</xsl:template>

<xsl:call-template name="greet">
  <xsl:with-param name="who" select="'world'"/>
</xsl:call-template>

Recursion: doing a thing N times

In 1.0 there is no for i = 1 to n. To process each character of a string you write a template that handles the first character and then calls itself on the rest — stopping at a base case. Here's a string reverser:

string-tools.xsl
<xsl:template name="reverse-string">
  <xsl:param name="s"/>
  <xsl:if test="string-length($s) > 0">
    <!-- emit the LAST character... -->
    <xsl:value-of select="substring($s, string-length($s), 1)"/>
    <!-- ...then reverse everything before it -->
    <xsl:call-template name="reverse-string">
      <xsl:with-param name="s" select="substring($s, 1, string-length($s) - 1)"/>
    </xsl:call-template>
  </xsl:if>    <!-- empty string = base case, recursion stops -->
</xsl:template>

⚑ The two-part recipe for any recursion

1. Base case — when do you stop? (Here: the string is empty.) 2. Reduce & recurse — do a bit of the work, then call yourself with a smaller input that moves toward the base case. Get these two right and the recursion terminates. Miss the base case and you get an infinite loop.

ⓘ 2.0 makes most of this unnecessary

With sequences and xsl:function you can often write a plain for-expression instead of hand-rolled recursion. But recursion is still the clearest way to express character-by-character algorithms — which is precisely why it's the backbone of the pure-XSLT ciphers ahead.

★ Practical use case — building strings the data doesn't give you

Recursion generates content that isn't in the source: repeat a character to draw a progress bar, indent nested items, split a comma-list into elements, pad a number to a fixed width for a fixed-format export. Any “do this until…” over text is a named recursive template.

Your turn

Sketch a named template repeat that outputs a string $s exactly $n times. What are its base case and its reduction?

Show a solution 
<xsl:template name="repeat">
  <xsl:param name="s"/>
  <xsl:param name="n"/>
  <xsl:if test="$n > 0">
    <xsl:value-of select="$s"/>
    <xsl:call-template name="repeat">
      <xsl:with-param name="s" select="$s"/>
      <xsl:with-param name="n" select="$n - 1"/>
    </xsl:call-template>
  </xsl:if>
</xsl:template>

Base case: $n = 0. Reduction: emit once, recurse with $n - 1.
