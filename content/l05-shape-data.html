Copying data across is only half the job. Real transforms reshape it: reorder rows, branch on conditions, and compute attribute values on the fly. Folder: examples/04-shape-data.

Sorting with xsl:sort

Drop an xsl:sort inside a for-each (or apply-templates) and the selected nodes are reordered before processing.

report.xsl · key parts
<xsl:for-each select="book">
  <xsl:sort select="price" data-type="number" order="descending"/>

  <tr class="{@category}">                       <!-- attribute value template -->
    <td><xsl:value-of select="title"/></td>
    <td><xsl:value-of select="concat(price/@currency, ' ', price)"/></td>
    <td>
      <xsl:choose>
        <xsl:when test="@inStock='true'">in stock</xsl:when>
        <xsl:otherwise>backorder</xsl:otherwise>
      </xsl:choose>
    </td>
  </tr>
</xsl:for-each>

Without data-type="number", xsl:sort compares as text and 17.99 sorts after 149.85 — a classic bug.

Branching: if and choose

  • xsl:if — a single yes/no test, no else.
  • xsl:choose / xsl:when / xsl:otherwise — multi-way, like switch/case.

Attribute Value Templates (AVTs)

Inside an attribute, curly braces { } switch on XPath. class="{@category}" evaluates @category and drops the result in — so a <book category="tech"> becomes <tr class="tech">. AVTs work in any literal attribute.

$ xsltproc 04-shape-data/report.xsl data/catalog.xml
<table border="1"> <tr><th>Title</th><th>Author</th><th>Price</th><th>Status</th></tr> <tr class="tech"> <td>XSLT 2.0 and XPath 2.0</td><td>Michael Kay</td> <td>USD 59.99</td><td>backorder</td> </tr> <tr class="security"> <td>Real-World Cryptography</td><td>David Wong</td> <td>USD 59.99</td><td>in stock</td> </tr> … (rows continue, most-expensive first) … <tr class="fiction"> <td>Cryptonomicon</td><td>Neal Stephenson</td> <td>USD 17.99</td><td>backorder</td> </tr> </table>

★ Practical use case — the invoice/report generator

This is the everyday XSLT job: turn a data feed into a styled document. Sort line items by amount, colour overdue rows red with an AVT (class="{if(daysLate>0,'late','ok')}" in 2.0), print “PAID” vs “DUE” with a choose. The capstone builds exactly this for a batch of orders.
