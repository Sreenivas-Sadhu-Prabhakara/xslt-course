Stylesheet skeleton

the shape of every stylesheet
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0"
                xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="html|xml|text" indent="yes"/>
  <xsl:template match="/"> ... </xsl:template>
</xsl:stylesheet>

Run a transform

by processor
# XSLT 1.0 — xsltproc (ships with macOS/Linux)
xsltproc style.xsl input.xml
xsltproc --stringparam name value style.xsl input.xml

# XSLT 2.0 / 3.0 — Saxon-HE (java)
java -jar Saxon-HE-10.9.jar -xsl:style.xsl -s:input.xml name=value

# XSLT 1.0 — Python / lxml
python3 -c "from lxml import etree as e; \
print(e.XSLT(e.parse('style.xsl'))(e.parse('input.xml')))"

Version quick-pick

1.0

Only if forced

Browsers, legacy stacks. No functions, no regex.

2.0

Sane baseline

Functions, regex, grouping, types.

3.0

Modern default

Maps, JSON, streaming, higher-order fns.

Gotchas that cost an afternoon

  • Selecting an attribute? It needs @. book[category] ≠ book[@category].
  • Numbers sorting wrong? Add data-type="number" to xsl:sort.
  • Empty output? A built-in template may be swallowing content — add a matching rule.
  • Stray whitespace? Use xsl:text and xsl:strip-space.
  • XML comments can't contain --. Don't paste --flags into a comment.
  • Security: never hard-code keys in a stylesheet; disable DTD/external entities (XXE).
