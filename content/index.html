The whole language, in one picture

Every XSLT program is the same shape: a source tree goes in, a stylesheet of rules is applied, and a result tree comes out. Hold on to these three colours — you'll see them on every code block in the course.

Source · XML 
<book>
  <title>Learning XSLT</title>
  <author>M. Fitzgerald</author>
</book>
▸
Stylesheet · XSL 
<xsl:template match="book">
  <li>
    <xsl:value-of select="title"/>
  </li>
</xsl:template>
▸
Result · HTML 
<li>Learning XSLT</li>

Run everything yourself

The lessons aren't just to read. A sibling examples/ folder holds a runnable file for (almost) every lesson, each one executed and verified.

terminal
cd examples
./fetch-saxon.sh      # once, for the 2.0/3.0 examples (needs Java)
./run.sh 02           # run any single lesson's example
./run.sh all          # run them all, printing each command

You need xsltproc (ships with macOS/Linux) for the 1.0 examples, Java for Saxon-HE, and python3 with lxml + cryptography for the two real-crypto lessons.

⚠ The one rule that matters most

XSLT does not do cryptography. Lessons 10's ciphers are for learning, not for protecting anything real. When data must actually be safe, XSLT's job is to decide what and where — a vetted crypto library does the how (lessons 11–12). This course keeps that line bright.
