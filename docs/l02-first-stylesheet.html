TRANSFORMATION · LESSON 02

Your first stylesheet

Write, run, and understand a complete transform from XML to HTML in a dozen lines.

Enough theory — let's transform something. We'll turn the course's sample catalogue into an HTML page, then read the stylesheet line by line. Everything here runs; the folder is examples/01-first-transform.

The input

A trimmed version of examples/data/catalog.xml:

catalog.xml · source
<catalog>
  <book id="b1" category="tech">
    <title>Learning XSLT</title>
    <author>Michael Fitzgerald</author>
  </book>
  <!-- ...five more books... -->
</catalog>

The stylesheet

to-html.xsl · stylesheet
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0"
                xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html" indent="yes"/>

  <xsl:template match="/">
    <html>
      <body>
        <h1>Bookshop catalogue</h1>
        <p>We have <xsl:value-of select="count(catalog/book)"/> titles.</p>
        <ul>
          <xsl:for-each select="catalog/book">
            <li>
              <xsl:value-of select="title"/> — <xsl:value-of select="author"/>
            </li>
          </xsl:for-each>
        </ul>
      </body>
    </html>
  </xsl:template>

</xsl:stylesheet>

Line by line

PartWhat it does
xsl:stylesheetThe root. version and the xsl: namespace URI are mandatory — that exact URI is how a processor recognises XSLT.
xsl:outputDeclares the result kind (html, xml, or text) and formatting. Optional but polite.
xsl:template match="/"A rule for the document root /. Its body describes the whole output.
literal <html>, <li>…Literal result elements: any element not in the xsl: namespace is copied straight to the output.
xsl:value-ofEvaluates an XPath and inserts the resulting text.
xsl:for-eachProcesses each selected node in turn. (A loop — handy, though you'll often prefer template rules; see Lesson 4.)

The key idea: your stylesheet is a skeleton of the output (the literal elements) with xsl: instructions sprinkled in wherever data must be pulled from the source.

Run it

$ xsltproc 01-first-transform/to-html.xsl data/catalog.xml
<html> <body> <h1>Bookshop catalogue</h1> <p>We have 6 titles.</p> <ul> <li>Learning XSLT — Michael Fitzgerald</li> <li>XSLT 2.0 and XPath 2.0 — Michael Kay</li> <li>XML in a Nutshell — Elliotte Rusty Harold</li> <li>Serious Cryptography — Jean-Philippe Aumasson</li> <li>Real-World Cryptography — David Wong</li> <li>Cryptonomicon — Neal Stephenson</li> </ul> </body> </html>

ⓘ whitespace & xsl:text

XSLT collapses insignificant whitespace between elements but keeps text you emit. When spacing matters, use <xsl:text> — </xsl:text> to say exactly what you mean — you'll see this throughout the course's text outputs.

★ Practical use case — nightly catalogue export

An online shop keeps its inventory as XML. A cron job runs this stylesheet each night to regenerate a static HTML price list for the warehouse — no database, no app server, just xsltproc feed.xml in a shell script. Swap the stylesheet and the same feed becomes a supplier CSV or a sitemap.

Your turn

Change the stylesheet so each <li> also shows the price, like “Learning XSLT — Michael Fitzgerald ($39.95)”.

Show a solution

Add another value-of inside the <li>:

<li>
  <xsl:value-of select="title"/> — <xsl:value-of select="author"/>
  (<xsl:value-of select="price"/>)
</li>

Because price is a child of the current book, the plain relative path price just works.
