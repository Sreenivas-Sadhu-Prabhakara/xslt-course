TRANSFORMATION · LESSON 06

Keys & grouping

Fast lookups with keys, and turning a flat list into groups — the 1.0 way and the 2.0 way.

Two related jobs come up constantly: looking things up fast, and grouping a flat list into buckets. XSLT 1.0 does both with xsl:key — grouping via a famous trick — and 2.0 adds a dedicated instruction that makes grouping trivial. Folder: examples/05-keys-grouping.

Keys: an index for your document

xsl:key builds a lookup table once, so key('name', value) retrieves matching nodes instantly instead of re-scanning the tree.

declare + use a key
<xsl:key name="byCategory" match="book" use="@category"/>
...
<xsl:value-of select="count(key('byCategory', 'security'))"/>   → 2

Grouping in 1.0: the Muenchian method

XSLT 1.0 has no “group by”, so we use a key to find, for each category, the first book — and treat that representative as the group header.

muenchian-1.0.xsl
<xsl:for-each select="book[generate-id() =
                           generate-id(key('byCategory', @category)[1])]">
  <xsl:sort select="@category"/>
  <xsl:value-of select="@category"/> (<xsl:value-of
      select="count(key('byCategory', @category))"/>)
  <xsl:for-each select="key('byCategory', @category)">
    <xsl:sort select="title"/>
    ...one line per book...
  </xsl:for-each>
</xsl:for-each>

It works — and it's notoriously hard to love. generate-id() gives each node a unique id; a book is the group's representative when its id equals the id of the first node the key returns for that category.

Grouping in 2.0: xsl:for-each-group

The same result, without the ceremony:

grouping-2.0.xsl
<xsl:for-each-group select="book" group-by="@category">
  <xsl:sort select="current-grouping-key()"/>
  <xsl:value-of select="current-grouping-key()"/>
  (<xsl:value-of select="count(current-group())"/>)
  <xsl:for-each select="current-group()"> ... </xsl:for-each>
</xsl:for-each-group>

Both stylesheets produce identical output:

$ xsltproc 05-keys-grouping/muenchian-1.0.xsl data/catalog.xml
Books grouped by category fiction (1): - Cryptonomicon security (2): - Real-World Cryptography - Serious Cryptography tech (3): - Learning XSLT - XML in a Nutshell - XSLT 2.0 and XPath 2.0

⚑ The takeaway

If you're on a 2.0+ processor, always reach for xsl:for-each-group (it also does grouping by adjacency and by starting/ending markers). Learn the Muenchian method only because you will meet it in legacy 1.0 stylesheets and need to read it.

★ Practical use case — summaries & deduplication

Grouping powers every “group by” report: sales by region, log lines by error code, orders by customer. The same representative-node trick also deduplicates — keep only the first node per key value — which is how people collapse repeated records in a 1.0 pipeline that can't be upgraded.

Your turn

Using the 2.0 stylesheet, group the books by inStock instead of category. What's the one change?
